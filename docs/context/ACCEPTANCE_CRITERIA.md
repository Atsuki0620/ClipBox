# 受け入れ条件（FastAPI + Next.js 移行）

現行 Streamlit 版の各画面について、「**FastAPI + Next.js 版でもこれが満たされれば移行完了**」と
みなせる合否基準を日本語で記述する。テストコードは書かない。各項目は手動確認のチェックリスト。

> **副作用に関する共通の注意**
> ファイルが存在しない動画に対する操作は、再生・判定とも当該動画を `is_available=0`（利用不可）に
> 更新する。視聴履歴・判定履歴・レベル・ファイル名は不変（`core/video_manager.py:258, :326`）。
> 異常系の合否基準はこの挙動に合わせる。

---

## Tier 1 — ライブラリ

### 正常系
- [ ] レベルフィルタを変更すると、一致する動画のみ表示される
- [ ] 登場人物フィルタを選択すると、該当動画のみ表示される
- [ ] 保存場所フィルタ（C_DRIVE / EXTERNAL_HDD）を変更すると、該当動画のみ表示される
- [ ] キーワード検索すると、本質的ファイル名に部分一致する動画のみ表示される
- [ ] ソートを切り替えると、表示順が変わる（作成日 / レベル / 視聴回数 / 最終視聴 / タイトル）
- [ ] ページング（page_size）を変えると表示件数が変わり、ページ移動できる
- [ ] 再生ボタンを押すと外部プレイヤーが起動し、視聴回数が +1 される
- [ ] 判定を確定するとファイルがリネームされ、レベルバッジが変わる
- [ ] いいねボタンを押すといいね数が +1 される
- [ ] セレクション除外フィルタを有効にすると、選別対象・選別済み動画が一覧から消える

### 異常系
- [ ] ファイルが存在しない動画の再生ボタンを押すとエラーが表示され、視聴履歴は記録されず、可用性が「利用不可」に更新される
- [ ] ファイルが存在しない動画の判定を確定するとエラーが表示され、レベル・ファイル名・判定履歴は不変のまま、可用性のみ「利用不可」に更新される

### KPI
- [ ] 未判定数・判定済み数・判定率・本日の判定数が正しい値で表示される
- [ ] 判定後に KPI が更新される

## Tier 1 — ランダム

### 正常系
- [ ] 未判定動画がランダムに指定本数表示される
- [ ] シャッフル操作で表示される動画群が入れ替わる
- [ ] 各カードから再生・判定・いいねができ、視聴回数 / レベル / いいね数が反映される

### 異常系
- [ ] 未判定動画が0件のとき、空状態が表示されエラーにならない
- [ ] 表示中の動画が判定済みになると、再表示時には未判定一覧から外れる

### KPI
- [ ] 未判定数・判定率が表示され、判定後に更新される

---

## Tier 1 — 運命の1本

### 正常系
- [ ] 「運命の1本」を引くと、未判定動画が**純ランダム**に1本選ばれて表示される
- [ ] 引いた動画から再生・判定・いいねができる
- [ ] 引き直すと別の動画が選ばれうる

### 異常系
- [ ] 未判定動画が0件のとき、「対象なし」の表示になりエラーにならない

---

## Tier 2 — ライブラリ

### 正常系
- [ ] セレクションフォルダを指定すると、そのフォルダ配下の動画のみ表示される
- [ ] 状態フィルタ「すべて / 未選別のみ / 選別済みのみ」を切り替えると表示が変わる
- [ ] キーワード検索・ソート・ページングが機能する
- [ ] 再生・判定・いいねが機能し、各値が反映される
- [ ] 選別を確定すると「選別済み」表示（`+` プレフィックス）に変わる

### 異常系
- [ ] ファイル不在の動画の再生・判定は、エラー表示 + 可用性「利用不可」更新（履歴・レベル不変）
- [ ] 未接続のフォルダを指定した場合、空状態かエラーメッセージが表示される

### KPI
- [ ] 未選別数・判定済み数・判定率・本日の判定数が表示され、判定後に更新される

---

## Tier 2 — ランダム

### 正常系
- [ ] セレクションフォルダ内の未選別動画がランダムに表示される
- [ ] シャッフルで表示が入れ替わる
- [ ] 各カードから再生・判定・いいねができる

### 異常系
- [ ] 未選別動画が0件のとき、空状態が表示される

---

## Tier 2 — 運命の1本

### 正常系
- [ ] 「運命の1本」を引くと、未選別動画から**最終視聴日からの経過日数で重み付け**して1本選ばれる
- [ ] 長期間見ていない動画ほど選ばれやすい
- [ ] 引いた動画から再生・判定・いいねができる

### 異常系
- [ ] 未選別動画が0件のとき、「対象なし」が表示される

## ランキング

### 正常系
- [ ] 種別（視聴回数 / 視聴日数 / いいね / 総合）を切り替えると順位が変わる
- [ ] 期間（180日 / 1年 / 全期間）を切り替えると集計対象が変わる
- [ ] 最低レベルフィルタ（制限なし / Lv3+ / Lv4のみ）が反映される
- [ ] 表示件数（10 / 20 / 50）が反映される
- [ ] 利用可否フィルタで利用不可動画の表示/非表示が切り替わる
- [ ] 各ランキング項目から再生・判定・いいねができ、スコアが反映される

### 異常系
- [ ] 対象データが0件のとき、空状態が表示される

---

## 分析ダッシュボード

### 正常系
- [ ] 期間プリセット（全期間 / 直近7日 / 30日 / 90日 / 180日 / カスタム）で集計範囲が変わる
- [ ] カスタム期間で開始日・終了日を指定できる
- [ ] スコープ（利用可能のみ / 利用不可のみ / すべて）と論理削除の包含切替が反映される
- [ ] 視聴トレンド・判定トレンドのチャートが期間に応じて描画される
- [ ] 視聴回数 / 視聴日数 / いいねの各ランキングが表示される
- [ ] レベル分布・保存場所統計・ファイルサイズ分布・視聴回数分布が表示される
- [ ] 判定応答時間ヒストグラムが表示される
- [ ] セレクション判定トレンド・選別レベル分布が表示される

### 異常系
- [ ] 対象期間にデータがないとき、空のチャート/メッセージが表示されエラーにならない

---

## 検索

### 正常系
- [ ] キーワード検索で、全角半角・大文字小文字・カタカナ/ひらがなの差を吸収して一致する動画が表示される
- [ ] 保存場所（C_DRIVE / EXTERNAL_HDD）の絞り込みが反映される
- [ ] 検索結果から再生・判定・いいねができる

### 異常系
- [ ] 一致が0件のとき、空状態が表示される

---

## AVP 再生（並列再生）

> Phase 4-C で FastAPI + Next.js に移植。AVP 起動先は FastAPI 実行マシン。

### 正常系
- [ ] 各タブの動画カードでチェックを入れると、AVP 選択リストに追加される（最大4本）
- [ ] AVP 起動操作で、選択した動画が並列再生される
- [ ] AVP 起動自体では視聴履歴が増えない
- [ ] 再生後、各動画の評価カードから判定・いいねができる
- [ ] 選択リストのクリアで選択状態がリセットされる

### 異常系
- [ ] 5本以上は選択できない、または超過時にエラー/制限が表示される
- [ ] ファイル不在の動画を含む場合、AVP 起動が中止され日本語エラーが表示される

---

## 設定

### 正常系
- [ ] ライブラリルート・セレクションフォルダ・デフォルトプレイヤー・AVP実行ファイルパスを設定・保存できる
- [ ] 保存後に再読込しても設定が保持される
- [ ] セレクションフォルダの手動スキャンを実行でき、見つかった件数が表示される
- [ ] ライブラリスキャンを実行すると、保存済みルートを対象にスキャンされ、見つからなかった動画が利用不可になる

### 異常系
- [ ] 全ドライブ未接続でスキャン0件のときは、既存の可用性が誤って一括「利用不可」化されない（安全ガード）
- [ ] 不正なパスを保存しようとした場合、エラーが表示される

### バックアップ
- [ ] バックアップ作成を実行すると DB のコピーが生成され、ファイル名とサイズが表示される
