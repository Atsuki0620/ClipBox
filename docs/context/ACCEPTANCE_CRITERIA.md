# 受け入れ条件（FastAPI + Next.js 移行）

現行 Streamlit 版の各画面について、「**FastAPI + Next.js 版でもこれが満たされれば移行完了**」と
みなせる合否基準を日本語で記述する。テストコードは書かない。各項目は手動確認のチェックリスト。
Phase 5 で Streamlit 旧 UI を archive へ移す判断では、本書の全画面シナリオ完了を前提条件として扱う。
Phase 5 の詳細条件は `PHASE5_STREAMLIT_ARCHIVE.md` を参照。

> **副作用に関する共通の注意**
> ファイルが存在しない動画に対する操作は、再生・判定とも当該動画を `is_available=0`（利用不可）に
> 更新する。視聴履歴・判定履歴・レベル・ファイル名は不変（`core/video_manager.py:258, :326`）。
> 異常系の合否基準はこの挙動に合わせる。

## Tier 1 — ライブラリ

### 正常系
- [ ] レベルフィルタを変更すると、一致する動画のみ表示される
- [ ] 判定状態フィルタ「すべて / 未判定 / 判定済み」を切り替えると表示が変わる（未判定=レベル-1 / 判定済み=Lv0..4）
- [ ] 保存場所フィルタ（C_DRIVE / EXTERNAL_HDD）を変更すると、該当動画のみ表示される
- [ ] キーワード検索すると、本質的ファイル名に部分一致する動画のみ表示される
- [ ] ソートを切り替えると、表示順が変わる（作成日 / レベル / APP視聴回数 / 最終APP視聴 / タイトル）
- [ ] ページング（page_size）を変えると表示件数が変わり、ページ移動できる
- [ ] 再生ボタンを押すと外部プレイヤーが起動し、APP視聴回数が +1 される
- [ ] 判定を確定するとファイルがリネームされ、レベルバッジが変わる
- [ ] いいねボタンを押すといいね数が +1 される
- [ ] **Tier1 はセレクション関連（`!`/`+`）を常に表示しない**（exclude_selection 固定。選別は Tier2 の責務）
  - ※ 登場人物（performer）フィルタは廃止（フォルダ名由来の暫定抽出でフィルタに使えていなかったため）

### 異常系
- [ ] ファイルが存在しない動画の再生ボタンを押すとエラーが表示され、視聴履歴は記録されず、可用性が「利用不可」に更新される
- [ ] ファイルが存在しない動画の判定を確定するとエラーが表示され、レベル・ファイル名・判定履歴は不変のまま、可用性のみ「利用不可」に更新される

### KPI
- [ ] 未判定数・判定済み数・判定率・本日の判定数が正しい値で表示される
- [ ] 判定後に KPI が更新される

## あとで見る

### 正常系
- [ ] サイドバーから `/watch-later` に遷移できる
- [ ] `watch_later=1` の動画が「未処理」「確認・見直し」「処理済み候補」の3セクションに分かれて表示される
- [ ] 未処理には Tier1 未判定と Tier2 未選別が表示される
- [ ] 確認・見直しには処理済み状態だが最終 APP 再生日の無い動画が表示される
- [ ] 処理済み候補には処理済み状態かつ最終 APP 再生日のある動画が表示される
- [ ] 各カードで再生・レベル変更・いいね・AVP候補・あとで見る解除ができる
- [ ] あとで見るを手動解除すると、対象動画が `/watch-later` の一覧から外れる
- [ ] 処理済み候補の一括解除で、対象動画が `/watch-later` の一覧から外れる
- [ ] 判定/選別完了、処理済み動画へのいいね、処理済み動画の AVP 起動成功であとで見るが自動解除される
- [ ] 通常再生、未判定動画へのいいね、Tier2未選別動画へのいいね、AVP候補追加、AVP再生対象チェックだけではあとで見るが解除されない

### 異常系
- [ ] `watch_later=1` の動画が0件のとき、各セクションが空状態になりエラーにならない
- [ ] 処理済み候補が0件のとき、一括解除ボタンは表示されない

## AVP 再生（並列再生）

> Phase 4-C で FastAPI + Next.js に移植。AVP 起動先は FastAPI 実行マシン。

### 正常系
- [ ] 対象画面の動画カードでチェックを入れると、AVP 候補に追加される（候補数の上限なし）
- [ ] AVP 画面で候補から再生対象を選択できる（最大4本）
- [ ] AVP 起動操作で、再生対象の動画が並列再生される
- [ ] AVP 再生成功時に視聴履歴が記録される（候補追加だけでは記録されない）
- [ ] 再生後、各動画の評価カードから判定・いいねができる
- [ ] 全候補クリアで候補と再生対象がリセットされる
- [ ] AVP 再生成功後、再生対象がクリアされる

### 異常系
- [ ] 再生対象は5本以上選択できない、または超過時にエラー/制限が表示される
- [ ] ファイル不在の動画を含む場合、AVP 起動が中止され日本語エラーが表示される

### 手動確認マトリクス（PR #30 / 実機必須）
> `subprocess.Popen([avp_exe_path, ...paths])` は AVP の CLI 仕様（複数パス引数の解釈）に依存するため、実機で確認し結果を残すこと。

- [ ] 1本再生で起動する
- [ ] 2本再生で2本とも開く
- [ ] 4本再生で4本とも開く（画面分割）
- [ ] 日本語・スペースを含むパスで正しく開く
- [ ] 外付けHDD上のパスで開く
- [ ] `avp_exe_path` 未設定 / 不正パスで日本語エラー（500）になる
