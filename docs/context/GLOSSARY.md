# ClipBox 用語集

対象読者: Coding agent。実装詳細ではなく、言葉がどういう意味で使われているかを定義する。
画面・状態の挙動仕様は `SPEC_NEXTJS.md`、DB は `DATA_MODEL.md` を正本とする（本書は用語の正本）。

## Phase 1 用語方針（2026-06-02）

この節を現行仕様の正本とする。

| 用語 | 現行の定義 |
|---|---|
| `!` prefix | セレクション未選別。`needs_selection = 1` かつ `is_selection_completed = 0` の対象を表す。 |
| `?` prefix | 旧資料・検討時の表記。現行コードでは使わない。新規 docs/code では `!` を使う。 |
| `+` prefix | セレクション完了。ファイル名上の完了 prefix。 |
| セレクション完了 | 実装・概念上の状態名。DB では `is_selection_completed`、ファイル名では `+` prefix。 |
| 選別済み | 画面表示用の短いラベル。概念としては「セレクション完了」と同じ状態を指す。 |
| ライブラリ取り込み済み | 現行仕様では使わない。物理的な取り込み・移動完了と、選別完了が混同されるため。 |
| レベル表記 | UI/docs は `未判定` / `Lv0` / `Lv1` / `Lv2` / `Lv3` / `Lv4` に統一する。 |
| `FAVORITE_LEVEL_NAMES` | レベル表示名の正本。表示名を変える場合はここを更新する。 |
| `performer` | DB・コード上のカラム名（動画の登場人物。親ディレクトリ名由来）。**Next.js 版では登場人物フィルタは廃止**（フォルダ名由来の暫定抽出が実用に耐えなかったため）。DB カラムは互換のため保持。 |
| `actors` | 旧 UI/session 名。active code では `performer(s)` に寄せる。 |
| 登場人物 | UI 日本語ラベル。DB/code の `performer` に対応。フィルタ機能としては廃止（上記）。 |
| `viewing_history` | 集計用の視聴履歴。視聴回数・ランキング・分析集計の基準。 |
| `play_history` | 再生ログ詳細。再生トリガー、プレイヤー、ライブラリルート、内部IDなどの詳細記録。 |

## 本質的ファイル名 (essential_filename)

プレフィックスをすべて除去した後のファイル名。
動画の同一性を判定する唯一の識別子。
パスが変わっても、プレフィックスが変わっても変化しない。
DB の UNIQUE 制約対象カラム。

---

## プレフィックス (prefix)

ファイル名の先頭に付くシステム管理文字列。お気に入りレベルとセレクション状態を表す。

| プレフィックス | 意味 |
|---|---|
| `####_` (Lv4) | Lv4 |
| `###_` (Lv3) | Lv3 |
| `##_` (Lv2) | Lv2 |
| `#_` (Lv1) | Lv1 |
| `_` (Lv0) | Lv0 |
| なし | 未判定（favorite_level = -1）|
| `!` | セレクション未選別（needs_selection = 1, is_selection_completed = 0）|
| `+` | セレクション完了（is_selection_completed = 1）。画面表示では「選別済み」。|

レベルプレフィックスとセレクションプレフィックスは組み合わせ可能（例: `!#_`）。

---

## お気に入りレベル (favorite_level)

動画の評価値。-1 から 4 の整数。

| 値 | 意味 |
|---|---|
| -1 | 未判定（プレフィックスなし） |
| 0 | Lv0（`_` プレフィックス） |
| 1〜4 | Lv1〜Lv4 |

DB カラム名: `current_favorite_level`

---

## 判定 (judgment)

動画に対してお気に入りレベルを設定する行為。
「判定する」= `set_favorite_level_with_rename()` を呼び出し、ファイルをリネームして DB を更新すること。
判定履歴は `judgment_history` テーブルに記録される。

## 判定済み (judged)

`current_favorite_level >= 0`（つまり 0, 1, 2, 3, 4 のいずれか）の動画。

## 未判定 (unrated)

**現行仕様**: Tier1 の未処理状態。`current_favorite_level = -1` の動画を指す。
UI 上では「未判定」と表示し、プルダウン等のレベル表示でも `Lv0` とは区別する。

補足:
- あとで見る運用では「未処理」に分類する。
- 判定が完了すると `current_favorite_level` が 0〜4 になり、あとで見るが ON の場合は現行実装で自動解除される。
- Tier2 文脈の未処理状態は「未選別」と呼び、「未判定」と混同しない。

---

## Tier 1（一次判定）

**現行仕様**: 動画の初期判定に関わる領域。未判定動画（`favorite_level = -1`）を対象に、初めてお気に入りレベルを付ける作業フロー。
ライブラリ・ランダム・運命の1本の3サブタブで構成する。

補足:
- Tier1 には「未判定」という状態がある。
- Tier1 未判定は、プルダウンなどの UI 上で「未判定」と表示する。
- Tier1 未判定の動画は、あとで見る運用では「未処理」として扱う。
- Tier1 未判定の動画を判定した場合、あとで見るが ON であれば自動解除される。

## Tier 2（二次判定）

**現行仕様**: Tier1 より後段の選別に関わる領域。セレクション動画（`!` プレフィックス付き）を対象に、選別作業を行うフロー。
ライブラリ・ランダム・運命の1本の3サブタブで構成する。

補足:
- Tier2 には「未選別」という状態がある。
- Tier2 未選別は UI 上で「未選別」と表示する。
- Tier2 未選別の動画は、あとで見る運用では「未処理」として扱う。
- Tier2 未選別の動画を選別した場合、あとで見るが ON であれば自動解除される。

設計方針:
- Tier2 で「未選別」と表示される場合、同じ情報を示す未選別バッジを重複表示しない方向で整理する。
- 現時点では `!###_` のような `!` 付きファイルが未選別を表す代表例だが、将来的に `!##_` や `!#_` も同じ未選別として扱う可能性がある。
- 未選別判定は、特定のプレフィックス文字列を各画面で決め打ちせず、共通ロジックへ集約する。

---

## セレクション (selection)

`!` プレフィックスが付いた動画群。一次判定済みの動画の中から選び出された、二次審査対象のグループ。
専用フォルダ（`selection_folder`）に物理的に移動された動画を管理する。

## 未選別 (needs_selection / unselected)

**現行仕様**: Tier2 の未処理状態。セレクション内で、まだ選別判定が行われていない動画。
`needs_selection = 1` かつ `is_selection_completed = 0` の状態。

補足:
- UI 上では「未選別」と表示する。
- あとで見る画面では「未処理」に分類する。
- 選別が完了すると `is_selection_completed = 1` になり、あとで見るが ON の場合は現行実装で自動解除される。

設計方針:
- 未選別バッジとプルダウン表示で同じ「未選別」を重複表示しない。
- 現時点の代表的な判定材料は `!###_` のような `!` プレフィックスだが、将来の拡張を想定する。
- 未選別判定は共通ロジックに集約し、画面・API・分析などで個別にプレフィックスを決め打ちしない。

## セレクション完了 / 選別済み (is_selection_completed)

セレクション内で、選別判定が完了した動画。
概念名は「セレクション完了」、画面表示は「選別済み」を使う。
DB カラム: `is_selection_completed BOOLEAN DEFAULT 0`

---

## 運命の1本 / 運命の一本 (fate video)

ボタン操作で動画プールから1本をランダム選出し、即座に再生・判定するモード。
Tier1/Tier2 とも既定は純ランダム選出。抽選結果は同じタブセッション内では `sessionStorage` に保持され、別画面へ移動して戻ってもカードを再表示する。復元表示では自動再生しない。

## 最近見てない優先 (recently_unwatched_priority)

運命の1本の任意トグル。ON のとき、最終視聴からの日数に応じて未再生・長く見ていない動画を少し出やすくする。重みは `weight = 1 + days / 90`、`days` は `0..180` に丸める。Tier1/Tier2 で別々に `user_config.json` の hidden fields に保存する。

---

## KPI

進捗を示す4つの指標。UI のメトリクスカードに表示される。

| 指標 | 意味 |
|---|---|
| 未判定数 (unrated_count) | `favorite_level = -1` の動画数 |
| 判定済み数 (judged_count) | `favorite_level >= 0` の動画数 |
| 判定率 (judged_rate) | 判定済み / 全動画 × 100 (%) |
| 本日の判定数 (today_judged_count) | 今日0:00以降に判定した動画数 |

セレクション KPI は別途: 未選別数 / 選別済み数 / 選別率 / 本日の選別数。

---

## スキャン (scan)

指定ディレクトリを走査して動画ファイルを DB に登録・同期する処理。
スキャン結果に含まれなかった動画は `is_available = 0` に更新される（論理的に「見つからなかった」扱い）。
ディレクトリが0件の場合はスキップ（安全ガード）。

---

## is_available

動画ファイルが現在アクセス可能かどうかを示すフラグ。
`1` = 利用可能（ファイルが存在）、`0` = 利用不可（ドライブ未接続・ファイル消失など）。
スキャンによって自動更新される。

## is_deleted

論理削除フラグ。`1` の動画はすべてのクエリから除外する。
物理ファイルは消えていても DB レコードは保持される。

---

## is_judging

再生開始から判定完了までの間、`1` にセットされる一時フラグ。
**Phase 1 でアーカイブ済み**: UI バッジ表示は無効化。DB カラムは保持。
→ `archive/video_manager_methods.py` 参照。

---

## 視聴履歴 (viewing_history)

動画を再生するたびに記録されるログ。テーブル: `viewing_history`。
`viewing_method` カラムで記録の種別（APP_PLAYBACK / FILE_ACCESS_DETECTED / MANUAL_ENTRY）を区別。

---

## スキャンディレクトリ (SCAN_DIRECTORIES)

スキャン対象のルートディレクトリ一覧。`config.py` の `SCAN_DIRECTORIES` と `user_config.json` の `library_roots` で管理。

## セレクションフォルダ (selection_folder)

セレクション動画を格納するフォルダパス。`user_config.json` の `selection_folder` で管理。
空文字の場合はフォルダ絞り込みなし（全セレクション動画を対象）。

---

## AVP (Awesome Video Player)

外部動画プレイヤー。`avp_exe_path` で実行ファイルパスを設定する。
最大4本を同時再生（画面分割）する並列再生機能を提供。

## デフォルトプレイヤー (default_player)

単体再生で使用するプレイヤー。`user_config.json` の `default_player` キー（デフォルト: `"vlc"`）。

---

## ライブラリルート (library_roots)

スキャン対象のルートディレクトリ一覧（`user_config.json`）。
スキャン時に `library_roots` → `SCAN_DIRECTORIES` の順にフォールバックする。

## ストレージ場所 (storage_location)

動画ファイルがどのドライブに存在するかを示す文字列。
値例: `"C_DRIVE"`, `"EXTERNAL_HDD"`。DB カラム: `storage_location`。

---

## カウンター A/B/C (counter)

**Phase 1 でアーカイブ済み。**
視聴回数を計測するためのタイマーカウンター。
A/B/C の3種類があり、いずれかが未開始の場合に全カウンターを一斉開始する。
→ `archive/counter_service.py` 参照。

## スナップショット (snapshot)

**Phase 1 でアーカイブ済み。**
ある時点のDB内容・フィルタ・設定を別SQLiteファイルに保存する機能。
→ `archive/snapshot.py` 参照。

## Next.js 版で追加された用語（2026-06）

挙動の正本は `SPEC_NEXTJS.md`。ここでは語の定義のみ固定する。

### クライアント状態 / サーバ状態
- **サーバ状態（DB永続）**: `videos.db` に保存され、全端末で一貫する状態。レベル・あとで見る・いいね・各履歴。
- **クライアント状態（localStorage永続）**: そのブラウザにのみ残る状態。AVP候補・AVP再生対象・再生中ハイライト。別ブラウザ/別PC/ストレージ削除で消える。
- フィルタ/ソート/ページはメモリのみ（永続なし。リロードで初期化）。

### あとで見る (watch_later)
**現行仕様**: 動画を後で判断・評価・活用するための一時キュー。単なる未視聴リストではない。
判定・選別を先延ばしするためのブックマークで、**DB 永続**（`videos.watch_later`、全端末共通）。

補足:
- 再生だけでは解除されない。
- 判定済み（level≥0）または選別完了（`+`）で自動解除される。
- AVP候補とは別物。あとで見るは「判断・評価・活用を後回しにする」ための状態、AVP候補は「AVPで再生する候補」のクライアント状態。

設計方針:
- あとで見る専用の横断一覧を新メニューとして作る。
- あとで見る画面では、付けた場所ではなく動画の状態ベースで分類する。
- 付けた場所は小さく補助表示する。
- 運命の1本由来は独立セクションにせず、動画の状態に応じて Tier1/Tier2 側に含める。
- 手動解除済み履歴は残さない。

### あとで見る画面
**予定仕様**: あとで見るに入れた動画を横断的に確認し、処理していくための専用画面。
ランキング画面内ではなく、新メニューとして作る。

設計方針:
- タスク消化専用画面として設計する。
- 主分類は動画の状態ベースにする。
- 付けた場所は補助情報として表示する。
- 画面構成は「未処理」「確認・見直し」「処理済み候補」のシンプル構成を基本とする。

分類例:
- **未処理**: Tier1 未判定、Tier2 未選別。
- **確認・見直し**: Tier1 判定済み、Tier2 選別済み。
- **処理済み候補**: あとで見る ON のままだが、現在状態から見て解除してもよさそうなもの。

### あとで見る自動OFF
**現行仕様**: 判定済み（level≥0）または選別完了（`+`）になった動画は、`watch_later` が自動解除される。
通常再生だけでは自動解除しない。

予定仕様:
- Tier1 未判定は、判定したら自動OFFする。
- Tier2 未選別は、選別したら自動OFFする。
- 判定済み・選別済みカードは、いいね、レベル変更、AVP再生で自動OFFする。
- 通常再生だけでは自動OFFしない。
- 自動OFF条件は、あとで見る画面だけでなく、どの画面で操作しても発動する。
- あとで見る画面は、自動OFF対象や解除候補を見やすくするタスク消化専用画面として設計する。

### 処理済み候補
**予定仕様**: あとで見る ON のままだが、現在状態から見て解除してもよさそうな動画群。

補足:
- 処理済み候補に入っただけでは自動OFFしない。
- ユーザーの一括解除操作により、あとで見るを OFF にする。
- 一括解除できるようにする。
- 初期の候補条件として、Tier1 判定済み、Tier2 選別済み、通常再生済みなどが考えられる。

### 確認・見直し
**予定仕様**: あとで見る画面内で、既に判定済み・選別済みだが、後から再確認したい動画を扱う分類。

補足:
- 未処理とは区別する。
- 通常再生だけではあとで見るを自動OFFしない。
- いいね、レベル変更、AVP再生をした場合は自動OFF対象になる。
- 何も変更しなかった場合は、処理済み候補または手動解除対象として扱う。

### AVP (Awesome Video Player)
**現行仕様**: 複数の動画候補を選び、最大4本を並列再生に活用するための機能。
外部動画プレイヤーとしての AVP 本体と、ClipBox 内の AVP候補・AVP再生対象を区別する。

設計方針:
- AVP選択状態バッチは不要とする。
- AVP候補はチェックボックスで表現する。
- チェックボックスの tooltip は「AVPで再生する候補に追加」とする。
- 通常再生と AVP再生は、あとで見る自動OFFの扱いが異なる。
- 通常再生だけではあとで見るを自動OFFしない。
- AVP再生した場合は、判定済み・選別済みカードのあとで見るを自動OFFする。

### AVP候補 (avpCandidateIds)
**現行仕様**: AVP で並列再生するためにプールした動画群。**上限なし**。**localStorage**（`clipbox-avp`、そのブラウザのみ）。
チェックボックスで追加・解除する。

### AVP再生対象 (avpPlayTargetIds)
**現行仕様**: AVP候補の中から、今回 AVP で実際に再生する動画。**最大4本**（`MAX_AVP_PLAY_TARGET=4`）。**localStorage**（`clipbox-avp`）。

### 通常再生
**現行仕様**: 動画カードの通常の再生ボタンから動画を再生する操作。
通常再生だけでは、あとで見るを自動解除しない。

補足:
- 通常再生は「確認しただけ」の可能性があるため、判断完了とはみなさない。
- 視聴履歴は記録されるが、あとで見るの解除条件とは別に扱う。

### AVP再生
**現行仕様**: AVPで選んだ候補を再生に活用する操作。通常再生よりも「選んで活用した」意味が強い。

予定仕様:
- 判定済み・選別済みカードでは、AVP再生した場合にあとで見るを自動OFFする。

### 再生中ハイライト (playback highlight)
**直近に再生中として扱う動画**の見た目強調。「視聴済み（過去再生）」の意味ではない。
単体再生=1本（`singlePlayingId`）、AVP再生=最大4本（`avpPlayingIds`）。次の再生まで保持。**localStorage**（`clipbox-playback`）。
有効期限が無いため実態は「最後に再生した動画」。

### 判定日時ソート (sort=judged_at)
`judgment_history.judged_at`（最新の判定日時）を基準にした並び替え。「最近判定した順」に確認するためのソート。Tier1/Tier2 共通。

### 総合スコア / 総合ランキング (composite)
**現行仕様**: ランキング種別の1つ。迷った時に動画を見るため、強い動画や候補動画を確認する主要な場所。
式: `round((視聴日数×1 + いいね×3) × (1 + 0.5×判定済み + 0.3×選別済み) × 100)`。
因子は視聴日数・いいね・Tier1判定済み・Tier2選別済み。視聴回数・レベル値・あとで見る・AVP候補は**スコアに入れない**。詳細・理由は `SPEC_NEXTJS.md` §9。

検討中:
- 総合ランキングに載る動画ばかりが再生・評価され、未露出動画との二極化が進む可能性がある。
- 総合ランキング自体を変更するか、別枠で発掘候補を作るかは、データ分析を行ってから判断する。
- 総合ランキング公平化は、まず分析用 notebook で傾向を確認する。
- 総合ランキング公平化は即時実装する仕様ではなく、分析後に方針決定する対象。

### 発掘候補
**検討中**: 総合ランキングにはまだ上がっていないが、有望そうな動画を見つけるための候補群。
総合ランキングとは別枠で検討する可能性がある概念で、まだ仕様確定していない。

候補条件の例:
- 再生機会が少ない。
- 判定機会が少ない。
- 部分的な評価が高い。
- 最近見ていない。

### 動画カード表示設定
**予定仕様**: 動画カードに表示する情報を全ページ共通で調整する設定。

設計方針:
- 設定画面で横断管理する。
- ページごとの個別設定にはしない。
- 全動画カード共通に適用する。
- 設定は保存し、再起動後も引き継ぐ。
- 表示 ON/OFF 対象は、ストレージ、ファイルサイズ、最終再生日、未判定/未選別、スコア、ファイル更新日、タイトル最大文字数。
- 動画ファイル名は表示 ON/OFF 対象から除外する。
- 動画ファイル名は、動画タイトルに hover したときだけ tooltip で表示する。
- 日付は `2026/06/12` 形式で表示する。

### 動画ファイル名
**設計方針**: 動画タイトルとは別に、実ファイル名を確認するための情報。

補足:
- 動画カード上には常時表示しない。
- 動画カード表示設定の ON/OFF 対象にはしない。
- 動画タイトルに hover したときだけ tooltip で表示する。

### 付けた場所
**予定仕様**: あとで見るを ON にしたときの画面または導線を示す補助情報。

補足:
- あとで見る画面では、主分類ではなく補助表示として扱う。
- 主分類は動画の状態ベースにする。
- 例として、Tier1、Tier2、ランキング、運命の1本などがある。
- 運命の1本由来は独立セクションにしない。
