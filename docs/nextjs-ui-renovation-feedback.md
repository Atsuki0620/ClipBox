## 全画面共通フィードバック

画面別フィードバックに先立ち、全画面に効く表示方針を記録する。各画面の改善要望はこの共通方針を前提にする。実装は今回行わない（docs-only 記録）。実装時には API で必要値が取れるか別途確認する。視聴履歴・DB・API・ランキング計算式は今回変更しない。

### 視聴指標は視聴日数に一本化する
- 視聴回数ではなく **視聴日数** に一本化する。動画カード・バッジ・ランキング・検索・AVP・あとで見る・設定サンプルカードなど全画面で、視聴回数表示を原則廃止し、視聴日数を標準表示にする。
- これは「視聴日数を優先表示する」という意味ではなく、「視聴回数の集計・表示を主役から外し、視聴日数に一本化する」という意味。ランキングの総合スコアで視聴日数を用いる方針（APP_PLAYBACK 基準）に全画面を揃える。
- 視聴履歴そのものは削除しない。視聴日数を計算する元データとして必要なため。

### 使う日付は 作成日 / 判定日 / 選別日 を基本にする
- **更新日・登録日は全 UI から外す**。カードやテーブルで使う日付は **作成日 / 判定日 / 選別日** を基本にする。

  | 日付 | 扱い | 表示名 |
  |---|---|---|
  | 作成日 | 動画ファイルの作成日（API 上の `file_created_at` を指す前提）。`created_at`（ClipBox 側の登録日時）とは別物。 | 設定・カードとも `作成日` |
  | 判定/選別日 | Tier1 の判定日 / Tier2 の選別日。設定項目名は1つにまとめる。 | 設定項目名 `判定/選別日` / カード表示は Tier1=`判定日`・Tier2=`選別日` |
  | 更新日 | **全 UI から外す**。判定・選別でファイル名が変わると動画ファイルの更新日も変わり得るため、「動画の中身が更新された日」か「ClipBox 上の判定・選別操作で変わった日」か区別できず誤解を招く。 | 表示しない |
  | 登録日 | **全 UI から外す**。ClipBox に取り込まれた日付は通常の運用判断では優先度が低く、`作成日` と混同しやすい。 | 表示しない |

- `作成日` は「動画ファイルの作成日」として扱い、`登録日` としては使わない。

### 該当Tierバッジを新設し初期表示にする
- どの Tier 文脈のカードかを示す **該当Tierバッジ** を新設し、全画面共通の状態ラベルとして扱う。バッジ表示項目の初期 ON にする。
- 表示は `Tier1` または `Tier2` の Tier 名だけにする。Lv・未判定・未選別・判定済み・選別済みは該当Tierバッジに書かない。
- Tier1 / Tier2 でバッジ色を分ける。具体色は統合 Variant K で全体テーマに合わせて決め、ここでは固定しない。
- レベルはカード内のレベル選択ボタンや Tier 列で分かるため、バッジに重複表示しない。

### バッジ初期表示
- カード系 UI のバッジ初期 ON は **該当Tier / あとで見る / 利用不可**。
- 選択肢として残すのは **該当Tier / あとで見る / 利用不可 / AVP候補 / 再生中**。
- 再生中: 原則としてバッジよりハイライトを優先する。行表示の場合は行ハイライト、カード表示の場合はカードハイライトを優先する。

### 利用不可の表示
- 利用不可バッジと薄表示を画面ごとに整合させる。
- ランキング / 検索のテーブルでは行全体の薄表示を優先する。カードではバッジとカード全体の薄表示を併用してよい。
- 利用不可の再生操作や再生対象追加は無効化する。

### Tooltip 方針
- 長い説明は画面タイトル横、または項目横の Tooltip に集約する。カード内やテーブル行内に同じ説明を繰り返さない。
- 主要ボタンは、文言だけで意味が分かる場合は Tooltip を増やしすぎない。
- 内部用語をユーザー向け文言に出しすぎない。ただし docs では DB / localStorage / API などの状態境界を正確に記録する。

### テーブルとカードの使い分け
- 数値比較や候補選択が中心の画面はテーブルを優先する: ランキング / 検索 / AVP候補。
- 個別動画の操作や状態確認が中心の画面はカードを優先する: Tier1 / Tier2 / あとで見る / AVP再生セット。
- 同じ概念を画面ごとに違う見た目で出しすぎない。カードとテーブルで、視聴日数 / 該当Tier / あとで見る / 利用不可 の扱いを揃える。

### Runtime control（共通方針）
- Runtime control は設定画面に入れない。サイドバー下部の開発支援部品として扱う。
- Streamlit 表示・ボタンは UI から削除する。FastAPI / Next.js の状態表示と `アプリを停止` ボタンに整理する。詳細は画面別フィードバックの [Runtime control](#runtime-control) 欄。

## 画面別フィードバック

各画面に **採用判断 / 未決事項 / 改善要望 / 不具合メモ** を記録する。未記入欄は UI LAB レビューや Variant K サンプル確認版の試用後に追記する。

### Tier1
- **採用判断**: variant-k 方向で確定。詳細な確定フィードバック（実装指示）は末尾の付録に収録。
- **未決事項**: 記録なし（追記する）。
- **改善要望**: 記録なし（追記する）。
- **不具合メモ**: 記録なし（追記する）。

### Tier2
- **採用判断**: 現時点で専用 UI を決め切らない。次の2案を比較し、統合 Variant K 前または Variant K 試作時に判断する。Tier2 独自要素は多くない想定のため、重い専用 UI LAB を作る前に軽い比較で判断する。

  | 案 | 方針 | 向いている場合 |
  |---|---|---|
  | 案1: Tier1 Variant K 流用案 | Tier1 Variant K のカード・操作部品・余白・グリッド・基本レイアウトをほぼ流用し、文言だけ Tier2 用に変える。実装負荷を抑え、ClipBox 全体の操作感を揃える。 | Tier2 独自の操作が少ない／Tier1 のカード・操作部品で違和感なく選別できる／画面ごとの作り分けより全体の一貫性を優先したい。 |
  | 案2: Tier2専用文言強め案 | カードや操作部品は Tier1 から流用しつつ、見出し・タブ説明・空状態・フィルタ文言・未選別/選別済みの説明・選別後の次アクション説明だけ Tier2 向けに強める。 | Tier2 の「選別」文脈が分かりにくい場合。 |

- **改善要望（現時点で確定すること）**:
  - カード/操作部品は原則 Tier1 から流用する。Tier2 専用にする可能性があるのは、見出し・空状態・フィルタ文言などの文章部分。
  - 該当Tierバッジは `Tier2` だけ表示する。該当Tierバッジには Lv や 未選別/選別済み を書かない。具体色は統合 Variant K で全体テーマに合わせて決める。
  - 未選別/選別済みの専用バッジは出さない。未選別/選別済みはレベルボタン・フィルタ・文脈で分かるようにする。
  - 初期フィルタは **未選別 + 選別済み** を表示する。フィルタ候補は **すべて / 未選別 / 選別済み** とし、既定は **すべて**（＝未選別と選別済みを両方表示）。
  - Tier1 と初期表示・操作感をできるだけ合わせる。
- **未決事項**: どこまで文言を専用化するか（案1で十分か、案2が要るか）。Tier2 を完全共通化すると「選別」文脈が弱くなる可能性があるため、試作で違和感を確認する。
- **不具合メモ**: Tier1 / Tier2、判定済み / 未判定、選別済み / 未選別 を混同しない。`displayContext="tier2"` の3値固定を守る。セレクションの個別スキャンは `scan_single_directory()`（全件 `is_available=0` 化を避ける）を崩さない。今回はdocs-only記録であり実装は行わない。

### あとで見る
- **採用判断**: 案B「付与理由・状態説明強化型」をベースにする。案Cの上部サマリー（作業台的な全体サマリー）は不要。現行どおり **未処理 / 確認・見直し / 処理済み候補** の3セクションを維持し、処理済み候補は一括解除よりも個別解除を基本にする。
- **未決事項**: 案Bを本体へ反映する場合の詳細（セクション見出しの余白、カード内の情報順、モバイル時の列数、処理済み候補の一括解除を残すかどうか）。ただし、PC幅では5列を基本候補にする。
- **改善要望**: 共通動画カードの流用を優先し、あとで見る専用にカード構造を増やしすぎない。サムネイル/画像枠は使わず、情報カードで揃える。
  - 付与理由と状態説明を強める。例: 「あとで見たい理由」「処理状況」「次にできる操作」をカード内で読み取れるようにする。
  - 状態文言は `ステータス: Tier1 未判定` / `ステータス: Tier1 判定済み` / `ステータス: Tier2 未選別` / `ステータス: Tier2 選別済み` 方向で固定する。「Tier2状態」のような別ラベルは使わない。
  - 自動解除/手動解除ルールの説明は、画面タイトル「あとで見る」右横の Tooltip 1箇所に集約する。カード内に同じ説明を繰り返さない。
  - Tooltip には、判定/選別完了や処理済み動画へのいいねでは自動解除されること、通常再生やAVP候補追加では解除されないこと、AVP再生でも自動解除しない方針を入れる。
  - カード内に「AVP再生済み・あとで見るは継続」のような注記は置かない。説明はタイトル横 Tooltip に集約する。
- **不具合メモ**: 既存仕様との境界を崩さない。あとで見るは DB、AVP候補/再生対象は localStorage に置く。`displayContext` は `tier1` / `tier2` / `avp` の3値固定で、あとで見る専用値を追加しない。AVP再生であとで見るを自動解除しない方針は現行実装との差分があるが、今回のフィードバック記録では実装しない。

### AVP
- **採用判断**: UI LAB 既存4案（A 左右分割 / B 上下分割 / C タブ分離 / D 上下分割・候補=上＋下段リッチカード）はそのまま採用しない。案Dを発展させた **案D改「上段候補テーブル / 下段再生セット2×2カード」** を採用候補とする。
  - 画面は2段構成にする。上段＝**AVP候補テーブル**（一覧性重視で候補を管理）、下段＝**今回AVPで再生する再生セット（最大4本）を2×2カードで配置**。
  - 採用理由: 候補は上段で一覧性を重視して管理したい／再生対象は最大4本なので下段で2×2カードとして見せたい／上段はランキング・検索で検討した操作付きテーブルの方向に揃え、数値比較と候補選択をしやすくする／下段は Tier1 ライブラリ動画カードと同じデザイン構成に寄せ ClipBox 全体の操作感を揃える／画面全体の文字量を減らし、説明や注意書きは画面タイトル横の Tooltip に集約する。
  - 再生対象の見せ方: スロット番号（`1 / 2 / 3 / 4`）は表示せず、配置順だけで表現する。空き枠を出す場合は控えめなプレースホルダにする。
- **未決事項**: 案D改の詳細設計（上段の列幅、モバイル時のテーブルの畳み方、下段2×2のレスポンシブ、空き枠プレースホルダの見せ方、一括いいねボタン名の確定）。総合スコア列に総合順位を表示するのに必要な値が既存 API で取れるか（実装時に確認し、不足する場合は別途仕様確認する。今回はdocs-only記録であり API 変更は行わない）。
- **改善要望（上段: AVP候補テーブル）**: カードではなく、ランキング/検索の操作付きテーブルと整合したテーブル表示にする。通常列は次の構成案。

  | 列 | 内容・方針 |
  |---|---|
  | タイトル | 省略表示。ファイル名全体は Tooltip で表示。動画ファイル名そのものをオン/オフ対象にしない既存方針を壊さない。 |
  | 総合スコア | ポイントと総合順位をまとめて表示（例: `123.4 pt（総合 12位）`）。順位の既定は **再生可能だけ** の総合ランキング順位、トグルで **全動画** の順位へ切替。表示範囲の考え方はランキング画面の `再生可能だけ / 全動画` と揃える。 |
  | 視聴日数 | 視聴日数を表示。ランキング公平化後の APP_PLAYBACK 基準と整合させる（計算ロジックは今回変更しない）。 |
  | いいね | `♡ 3` のように表示。上段は**表示のみでクリック不可**（いいね操作は下段の再生セットカードで行う）。 |
  | あとで | バッジではなく `あとで` 列として、あとで見る状態を表示。あとで見るが AVP候補とは別の印であることはタイトル横 Tooltip でも説明する。 |
  | Tier1 | `未判定 / Lv0〜Lv4 / —`。 |
  | Tier2 | `未選別 / Lv0〜Lv4 / —`。Tier2 の `Lv0〜Lv4` は選別済み扱い。 |
  | アクション | `再生対象に追加` / `候補から外す`。主要ボタン名で意味が分かるため個別 Tooltip は不要。 |

  - いいね列の役割分担: 上段は「候補を選ぶ場所」、下段は「今回再生する最大4本を仕上げる場所」。役割を分けて操作の意味を分かりやすくする。
  - 状態列の方針: Tier1 / Tier2 を `Tier1未判定 / Tier1判定済み / Tier2未選別 / Tier2選別済み` のように1列で混在表示しない。Tier1列・Tier2列を分ける。Tier1/Tier2・判定済み/未判定・選別済み/未選別を混同しない。`Tier2状態` のような別ラベルは増やさない。
  - バッジ方針: 候補テーブルでは基本的にバッジを出さない（文字量を減らし、ランキング/検索のテーブルと整合した数値中心の見え方に寄せる）。情報は 総合スコア / 総合順位 / 視聴日数 / いいね数 / あとで列 / Tier1列 / Tier2列 で表現する。
    - 利用不可: バッジではなく**行全体の薄表示**で示し、利用不可の候補は `再生対象に追加` を無効化する（ランキング/検索/Tier1 の「利用不可は薄表示」と整合）。
    - 再生中: バッジではなく**行ハイライト**で示す。
- **改善要望（下段: 再生セット2×2カード）**: 今回AVPで再生する最大4本を 2×2カードで表示する。
  - Tier1 ライブラリ動画カードと同じデザイン構成に寄せる。サムネイルや画像枠は使わず情報カード方針を維持し、ClipBox 全体のカードデザインと整合させる。可能な範囲で共通動画カード部品を使う。
  - カードの主な要素: タイトル / メタ情報 / Tier1・Tier2 のレベル表示または操作 / いいね / あとで見る / 再生 / 再生対象から外す操作。
  - 個別いいね: 下段カードで個別にいいねを押せる。挙動は既存 ClipBox 仕様に合わせ、AVP画面だけ独自のいいね挙動にしない（Tier1/Tier2/ランキング/検索と操作感を揃える）。

### 設定
- **採用判断**: UI LAB scan-first 系のうち **scan-first の上部タブ案** を採用候補とする（単一カラム版・カテゴリレール版は比較参考）。レポート: `frontend/src/app/lab/settings/_review/COMPARISON_SCAN_FIRST.md`。

- **改善要望（カード表示設定＝メタ項目とバッジ項目に分離）**: カード内の情報を **メタ項目**（タイトル下などに表示する補足情報）と **バッジ項目**（カード上に小さく表示する状態ラベル）の2種類に分けて扱う。

  | 区分 | 初期ON | 選択肢として残す | 全UIから外す |
  |---|---|---|---|
  | メタ項目 | ストレージ / 視聴日数 / 作成日 | ストレージ / 視聴日数 / 作成日 / 判定・選別日 / サイズ / 最終再生日 | 視聴回数 / 更新日 / 登録日 |
  | バッジ項目 | 該当Tier / あとで見る / 利用不可 | 該当Tier / あとで見る / 利用不可 / AVP候補 / 再生中 | — |

  - **メタ項目の詳細**:
    - 視聴日数: 視聴回数ではなく視聴日数を使う。ランキングの総合スコアで視聴日数を用いる方針にしたため、全画面で視聴日数へ一本化する（[全画面共通フィードバック](#全画面共通フィードバック) 参照）。視聴履歴そのものは削除しない。今回はdocs-only記録であり DB・API は変更しない。
    - 作成日: 設定画面では `作成日` と表示し、Tooltip で「動画ファイルの作成日」と補足する。API 上の `file_created_at` を指す前提。`created_at`（ClipBox 側の登録日時）と混同しやすいため、UI 上では `登録日` としても使わない。
    - 判定/選別日: 設定画面のトグル名は `判定/選別日` とする。実際のカード表示では Tier1 なら `判定日`、Tier2 なら `選別日` と表示する。初期 ON にはせず、選択肢として残す。
    - 並び順: 既定の並び順は `ストレージ / 視聴日数 / 作成日`。将来的にはメタ項目をドラッグ&ドロップで並び替えできるようにしたい。ドラッグ&ドロップ並び替えは設定保存が必要になるため、実装時は設定キー設計が必要。今回はdocs-only記録であり、設定キーや `user_config.json` は変更しない。
  - **バッジ項目の詳細**:
    - 該当Tierバッジ（新設）: レベルは書かず、`Tier1` または `Tier2` の Tier 名だけを表示する。Tier1 / Tier2 でバッジ色を分ける。具体色は統合 Variant K で全体テーマに合わせて決め、ここでは固定しない。理由: レベルはカード内のレベル選択ボタンや Tier 列で分かるためバッジに重複表示しない／該当Tierバッジは、どの Tier 文脈のカードかを示す印として使う。
    - あとで見るバッジ: あとで見る ON の動画に表示する。既存のあとで見る状態と整合させる。あとで見るは AVP候補とは別の状態として扱う。
    - 利用不可バッジ: 利用不可の動画に表示する。画面によってはカード全体の薄表示や行全体の薄表示も併用する。利用不可の扱いは Tier1 / Tier2 / あとで見る / AVP / ランキング / 検索で整合させる。
    - AVP候補バッジ: 選択肢として残し、初期 ON にはしない。AVP 画面や候補管理では、バッジよりも列や状態表示を優先する場合がある。
    - 再生中: 選択肢として残し、初期 ON にはしない。原則としてバッジよりもカードハイライトを優先する。
  - **レベル表示対象（バッジ表示項目とは別の設定）**: `該当Tierのみ` / `Tier1・Tier2を両方表示` の2択にする。既定値は `該当Tierのみ`。
    - 意味: `該当Tierのみ`＝Tier1 画面では Tier1、Tier2 画面では Tier2 だけを表示する／`Tier1・Tier2を両方表示`＝どの画面でも Tier1/Tier2 両方のレベル情報を表示する。
    - 注意: `全Tier` という表現は使わない（`Tier1・Tier2を両方表示` の方が何が起きるか具体的で分かりやすい）。該当Tierバッジとは別の設定として扱う。レベル表示対象＝レベルボタンやレベル表示をどこまで出すかの設定。バッジ表示項目＝該当Tier / あとで見る / 利用不可 などを出すかの設定。
- **未決事項**: 上部タブ版の細部（タブ並び、自動保存デバウンス、進捗表示の粒度）、メタ項目ドラッグ&ドロップの設定キー設計、メタ項目の `判定/選別日`・`作成日` 表示に必要な値が既存 API（`file_created_at` 等）で取れるか。実装時に確認し、不足する場合は別途仕様確認する。

## 2. 共通変更（サイドバー & 見出し & カード）

### 2-1. サイドバー（ラボ `ModernSidebar` 相当・variant-k のみ）

- **ロゴ追加**: 「ClipBox」の**左に簡素な自作SVGロゴ**を置く。

### 2-2. 見出し（3タブ共通）

- **タイトル2行表示**: `line-clamp-2` は維持しつつ、**2行ぶんの高さを常に確保**してカード高さを揃える（短いタイトルでもカードが不揃いにならないよう、タイトル領域に2行分の min-height を与える）。

## 3. タブ別変更

### 3-1. ライブラリ（`tier1-library/variant-k`）
